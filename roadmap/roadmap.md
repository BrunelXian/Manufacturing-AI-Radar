# Roadmap

## V0.1 - Foundational Structure

Objective: establish a clear and public-facing first version of the repository.

Key targets:

- rewrite `README.md`
- define repository architecture
- create manufacturing AI taxonomy
- create initial radar topic pages
- add maintenance and contribution guidance

## V0.2 - Topic Coverage Expansion

Objective: broaden and deepen topic coverage while preserving structure.

Key targets:

- complete remaining radar pages
- improve cross-links between related domains
- add richer examples and industrial framing
- strengthen the distinction between modelling, prediction, and control

## V0.3 - Papers, Tools, and Datasets Mapping

Objective: connect the radar structure to representative external resources.

Key targets:

- add curated paper maps by domain
- add representative tool and dataset summaries
- create lightweight annotation conventions for references
- distinguish foundational papers from emerging work

## V0.4 - Update Workflow and Automation

Objective: reduce maintenance friction and support recurring updates.

Key targets:

- define weekly or monthly review workflow
- add simple literature tracking routines
- create templates for new topic additions
- explore automation support for update reminders or scan tasks

## V1.0 - Stable Manufacturing AI Radar

Objective: form a coherent and extensible repository that can serve as a long-term reference point.

Key targets:

- stable taxonomy and domain coverage
- consistent topic pages across the main manufacturing AI landscape
- curated references, tools, and datasets
- practical maintenance logic for continued evolution

## Maintenance Logic

Future maintenance should follow four recurring actions:

1. Add new themes only when their scope is clear.
2. Add representative papers rather than exhaustive paper dumps.
3. Add tools and datasets with manufacturing context and task labels.
4. Review the repository on a weekly or monthly cadence to keep structure coherent.
