# Contribution Guide

## Purpose

This repository grows best when contributions improve structure, clarity, and long-term usefulness. Contributors should treat it as a curated research navigation project, not as a free-form collection of links.

## What Good Contributions Look Like

Strong contributions usually do one of the following:

- improve a radar domain page with clearer manufacturing context
- add representative papers with brief annotations and relevance
- add tools or datasets in a structured way
- refine taxonomy boundaries where confusion exists
- add industrial case examples that clarify real use

## Contribution Principles

- prefer structured additions over long unfiltered lists
- keep the manufacturing problem explicit
- explain why a method matters, not just what it is called
- use terminology consistently across files
- avoid duplicating content that belongs elsewhere in the repository

## How To Add a New Topic

When adding a new topic page:

1. Check whether the topic already fits an existing radar page.
2. If it needs a new page, name it with a concise lowercase hyphenated file name.
3. Follow the standard page structure:
   - What this domain covers
   - Why it matters in manufacturing
   - Typical tasks
   - Representative methods
   - Example application scenarios
   - Key research questions
   - Future opportunity
4. Link the page from `README.md` and related documents when appropriate.

## How To Add Papers

When adding representative papers in the future, include:

- title
- year
- manufacturing context
- problem addressed
- method category
- short note on why it is representative

Avoid adding raw paper dumps without explanation.

## How To Add Tools or Datasets

For tools and datasets, document:

- manufacturing domain
- input data type
- task type
- practical relevance
- maturity level or limitations

## Style Guidance

- write in concise professional English
- use Markdown headings consistently
- keep bullet lists focused and comparable
- avoid promotional language
- prefer explanation grounded in industrial context

## Maintenance Logic

If a contribution is exploratory and not yet stable, place it in `notes/` first and promote it later after refinement.
